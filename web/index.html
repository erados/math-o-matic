꿀팁. 이름을 정할 때 몇 가지 작명 방식을 따르고 있습니다.

  • T: verum. true의 T.
  • F: falsum. false의 F.
  • S: nand. sheffer stroke의 S.
  • N: not. not의 N.
  • A: and. and의 A.
  • O: or. or의 O.
  • I: implies. implies의 I.
  • E: iff. equivalent의 E.
  • V: forall. every의 V.
  • X: exists. exist의 X.
  • i: introduction. introduction의 i.
  • e: elimination. elimination의 e.
  • m: modus ponens. modus ponens의 m. |- p -> q 형태에 mp를 적용해서 p |- q 형태로 만들었을 경우 m을 붙입니다. |- p <-> q의 경우 p |- q가 m1이고 q |- p가 m2가 됩니다.
  • f: functional. functional의 f. Af나 Of처럼 함수 작용소 버전을 만들 때 붙입니다.
  • _def: 어떤 def의 정의로서 사용되는 rule의 이름의 경우 해당 def의 이름에 _def를 붙입니다. def emptyset과 rule emptyset_def의 관계를 참조하세요.
